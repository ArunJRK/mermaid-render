What ships today

Built for Mermaid projects that outgrow flat SVG

⌕

Interactive canvas

GPU-backed zoom, pan, fit, and reset for diagrams that need room.

▤

Node folding

Collapse subgraphs to keep large flows readable without leaving the page.

⇄

Cross-file links

Use allowlisted @link navigation across virtual .mmd projects in-browser.

◆

Two real layouts

Narrative and Blueprint are the v1 engines; other names are visual presets.

◈

WebGPU/WebGL path

Prefers WebGPU when available and falls back to WebGL when it is not.

≋

Mermaid-compatible

Flowchart Mermaid syntax stays the authoring format; interactivity is added at render time.

Mermaid in → canvas out

Plain source becomes a living diagram

source.mmd 
%% @link docs -> ./details#hydrate
graph TD
  syntax[Mermaid syntax] --> parse[Parse]
  parse --> layout{Narrative or Blueprint}
  layout --> canvas[GPU canvas]
  subgraph Project files
    docs[docs/architecture.mmd]
    canvas --> fold[Fold subgraphs]
  end

Embed API

Mount it anywhere you can provide a canvas

embed.ts 
import {
  MermaidRenderer,
  createVirtualFileResolver,
} from '@mermaid-render/core'

const canvas = document.querySelector('canvas')
if (!canvas) throw new Error('Missing canvas')

const files = {
  '/docs/root.mmd': `%% @link details -> ./details#hydrate
graph TD
  source[Mermaid] --> details[Interactive canvas]`,
  '/docs/details.mmd': `graph TD
  hydrate[Hydrate renderer] --> pan[Pan and zoom]`,
}

const renderer = new MermaidRenderer({ themeMode: 'system' })
const linkResolver = createVirtualFileResolver(files)

await renderer.mount(canvas)
await renderer.load(files['/docs/root.mmd'], {
  sourcePath: '/docs/root.mmd',
  linkResolver,
})
